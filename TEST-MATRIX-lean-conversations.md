# Lean-engine: симуляція 24 клієнтів (5-6 кроків кожен)

> Симуляція на папері за логікою `agent_lean.py`. Кожен крок: `DECIDE`(роут) →
> `QUERY`(query_prompt роута) → `TOOL`(сире) → `CLEAN`(голий виклик) → `ANSWER`.
> Реальні бізнес-факти: Львів, Івана Огієнка 15 · буд. 11-18, сб 11-16:30, нд —
> вихідний · тел. 067 385 15 60 · Нова Пошта 48 відд. (Кабан Андрій) · гарантія 60 днів.
> Роути тенанта: `catalog · qa · web_search · external_price · business_info · handoff`.
> **R!** = ризик/місце для перевірки наживо.

---

### 1. Не може знайти адресу
1. C: «привіт» → DECIDE: answer → A: «Привіт. Що з технікою?»
2. C: «а де ви?» → DECIDE: **business_info** → QUERY: `address` → TOOL: address+hours+phone → CLEAN: «Львів, Огієнка 15; буд 11-18; тел 067 385 15 60» → A: «Ми у Львові, Огієнка 15. Буд. 11-18, сб до 16:30.»
3. C: «а як доїхати від вокзалу» → DECIDE: answer (нема в базі маршруту) → A: «Маршрут не підкажу точно, але адреса Огієнка 15, телефонуйте 067 385 15 60.»
4. C: «о котрій сьогодні закрив.?» → DECIDE: **business_info** → QUERY: `hours` → CLEAN: «буд 11-18» → A: «Сьогодні до 18:00.»
5. C: «ок дякую» → DECIDE: answer → A: «Гарного дня!»
- R!: крок 3 не вигадати маршрут; крок 2 реальна адреса (не Київ).

### 2. Незрозумілий прилад → ремонт
1. C: «ремонтуєте Anbernic RG35XX?» → DECIDE: **web_search** → QUERY: `Anbernic RG35XX device type` → TOOL: «портативна ігрова консоль» → CLEAN: «тип: ретро ігрова консоль» → A: «Це портативна консоль — беремо. Що з нею?»
2. C: «не вмикається» → DECIDE: answer (тип відомий, симптом є) → A: «Схоже на живлення або АКБ, точно — після огляду. Привозьте на безкоштовну діагностику.»
3. C: «скільки діагностика» → DECIDE: **catalog** → QUERY: `діагностика` → CLEAN: «безкоштовна при ремонті» → A: «Діагностика безкоштовна при ремонті.»
4. C: «а де ви» → DECIDE: **business_info** → A: «Львів, Огієнка 15.»
5. C: «добре привезу» → DECIDE: answer → A: «Чекаємо, гарного дня.»
- R!: крок1 web визначає тип, не ціну; крок2 версія, не вердикт.

### 3. Ціна заміни дисплея з деталлю (КЛЮЧОВИЙ ланцюг)
1. C: «розбив екран на Xiaomi Redmi Note 10» → DECIDE: answer/catalog → A: «Так, міняємо дисплей. Хочете орієнтовну ціну?»
2. C: «так, скільки з деталлю» → DECIDE: **catalog** → QUERY: `заміна дисплея смартфон` → TOOL: «заміна дисплейного модуля (робота) 700-1800» → CLEAN: «робота 700-1800 грн (без деталі)» → факт збережено
3. (той самий тур, крок2 циклу) DECIDE: **external_price** → QUERY: `Xiaomi Redmi Note 10 LCD` → TOOL(web): ринкові 900-1500 → CLEAN: «дисплей ~900-1500 грн (ринок)»
4. ANSWER: «Робота 700-1800 грн, сам дисплей орієнтовно 900-1500 грн (ринок, купується окремо). Точно — після огляду.»
5. C: «а гарантія» → DECIDE: **qa**/business_info → CLEAN: «60 днів» → A: «Гарантія до 60 днів.»
- R! ВИСОК.: дві стадії (catalog→external_price) за один тур; розділ робота/деталь; не злити в одне число.

### 4. Ціна акумулятора з заміною
1. C: «треба поміняти батарею на айфон 11» → DECIDE: catalog → QUERY: `заміна акумулятора смартфон` → CLEAN: «робота 600-1400»
2. (цикл) DECIDE: external_price → QUERY: `iPhone 11 battery` → TOOL(web): ~700-1100 → CLEAN: «АКБ ~700-1100 (ринок)»
3. ANSWER: «Робота 600-1400 грн + АКБ ~700-1100 (ринок). Точно після огляду.»
4. C: «оригінал?» → DECIDE: answer → A: «По оригіналу скажемо на місці, залежить від наявності.»
5. C: «де ви» → business_info → A: «Львів, Огієнка 15.»
- R!: external_price тільки при явному запиті ціни + конкретна модель.

### 5. З іншого міста → Нова Пошта
1. C: «колонка JBL не заряджається» → DECIDE: answer → A: «Беремо JBL. Привозьте на діагностику.»
2. C: «я з Тернополя» → DECIDE: **business_info** → QUERY: `delivery` → CLEAN: «Нова Пошта, 48 відд., Кабан Андрій» → A: «Можна Новою Поштою: Львів, 48 відділення, Кабан Андрій.»
3. C: «а скільки ремонт» → DECIDE: catalog → QUERY: `ремонт колонки` → CLEAN: «300-3200 залежно» → A: «Орієнтовно 300-3200 грн залежно від несправності, точно — після огляду.»
4. C: «ок відправлю» → DECIDE: answer → A: «Чекаємо посилку, тел 067 385 15 60.»
- R!: Нова Пошта лише з business_info delivery; не вигадати відділення.

### 6. Неділя?
1. C: «ви в неділю працюєте?» → DECIDE: **business_info** → QUERY: `hours` → CLEAN: «нд вихідний; буд 11-18; сб 11-16:30» → A: «В неділю вихідний. Буд. 11-18, сб до 16:30.»
2. C: «а в суботу до котрої» → DECIDE: business_info → CLEAN: «сб до 16:30» → A: «До 16:30.»
3. C: «прийду в суботу о 12 з ноутом» → DECIDE: answer (графік ок) → A: «О 12 в суботу — заходьте, Огієнка 15.»
4. C: «дякую» → answer → A: «Гарного дня.»
- R!: перевірка дня проти графіка (нд відмова, сб ок).

### 7. Матриця ТВ (відмова за бізнес-правилом)
1. C: «розбита матриця на телевізорі Samsung 55"» → DECIDE: catalog/answer → A: «Матриці на ТВ не міняємо — невигідно, дешевше новий. Інші поломки ТВ беремо.»
2. C: «а підсвітку?» → DECIDE: catalog → QUERY: `ремонт підсвітки телевізора` → CLEAN: «заміна LED 55-65" 3200-6000 (робота)» → A: «Підсвітку робимо, орієнтовно 3200-6000 грн (робота), точно після огляду.»
3. C: «а на ноуті матрицю?» → DECIDE: catalog → CLEAN: «заміна матриці ноутбука (робота)» → A: «На ноуті матрицю міняємо, на відміну від ТВ.»
- R!: не сплутати правило ТВ-матриці з ноутом.

### 8. Блендер вмер
1. C: «вмер блендер» → DECIDE: catalog/answer → QUERY: `ремонт блендера` → CLEAN: «дрібна побутова, є категорія» → A: «Так, ремонтуємо. Привозьте на безкоштовну діагностику.»
2. C: «гудить але не крутить» → DECIDE: answer → A: «Схоже на муфту або редуктор/двигун, точно — після огляду.»
3. C: «скільки» → DECIDE: catalog → CLEAN: діапазон/після огляду → A: «Точно після діагностики, вона безкоштовна при ремонті.»
4. C: «де ви» → business_info → A: «Львів, Огієнка 15.»
- R!: версія причини, не вердикт; broad category appliances.

### 9. Кавомашина тече
1. C: «кавомашина DeLonghi протікає» → DECIDE: catalog → QUERY: `ремонт кавомашини` → CLEAN: «coffee-machines, є» → A: «Беремо кавомашини. Тече — привозьте, гляне майстер.»
2. C: «це дорого?» → DECIDE: catalog → CLEAN: діапазон/діагностика → A: «Залежить від причини, скажемо після діагностики (безкоштовна при ремонті).»
3. C: «я з Дрогобича» → business_info delivery → A: «Можна Новою Поштою, 48 відд., Кабан Андрій.»
4. C: «ок» → answer → A: «Чекаємо, гарного дня.»

### 10. Ноут не вмикається
1. C: «ноутбук не вмикається зовсім» → DECIDE: answer → A: «Схоже на живлення або плату, точно — після огляду. Привозьте.»
2. C: «заливав колись» → DECIDE: answer → A: «Після рідини буває корозія/КЗ. Не вмикайте, везіть якнайшвидше.»
3. C: «скільки чистка після залиття» → DECIDE: catalog → QUERY: `чистка ноутбука після рідини` → CLEAN: діапазон → A: «Орієнтовно <діапазон з каталогу>, точно після огляду.»
4. C: «де ви» → business_info → A: «Огієнка 15, Львів.»
- R!: safety після рідини; ціна лише на запит.

### 11. Навушники один бік мовчить
1. C: «airpods один не грає» → DECIDE: answer → A: «Беремо навушники. Може бути динамік або контакт — точно після огляду.»
2. C: «можна почистити?» → DECIDE: catalog → CLEAN: чистка → A: «Так, чистка можлива, привозьте.»
3. C: «скільки» → catalog → A: «<діапазон> / після огляду.»
4. C: «дякую» → answer → A: «Гарного дня.»

### 12. JBL не заряджається — деталь
1. C: «JBL Charge 4 не бере заряд» → DECIDE: answer → A: «Часто це роз'єм або контролер заряду. Привозьте.»
2. C: «скільки роз'єм поміняти» → DECIDE: catalog → QUERY: `заміна роз'єму зарядки колонки` → CLEAN: «робота 600-1200»
3. (цикл) DECIDE: external_price → QUERY: `JBL Charge 4 USB-C charging port` → CLEAN: «роз'єм ~150-400 (ринок)»
4. ANSWER: «Робота 600-1200 грн + роз'єм ~150-400 (ринок). Точно після огляду.»
- R!: ланцюг catalog→external_price для деталі.

### 13. «Чи є роз'єм для Samsung A52 і ціна з установкою»
1. C: «є роз'єм зарядки на Samsung A52?» → DECIDE: answer (ми не продаємо деталі окремо) → A: «Окремо запчастини не продаємо — ми сервіс. Можемо поставити в ремонті.»
2. C: «ну поставте, скільки» → DECIDE: catalog → QUERY: `заміна роз'єму зарядки смартфон` → CLEAN: «робота 700-1800»
3. (цикл) DECIDE: external_price → QUERY: `Samsung A52 charging port flex` → CLEAN: «деталь ~120-300 (ринок)»
4. ANSWER: «Окремо не продаємо, але в ремонті: робота 700-1800 + деталь ~120-300 (ринок). Точно після огляду.»
5. C: «де ви» → business_info → A: «Львів, Огієнка 15.»
- R! ВИСОК.: відмова продажу окремо + ціна з установкою (робота+деталь).

### 14. Гарантія
1. C: «яка у вас гарантія» → DECIDE: **qa**/business_info → QUERY: `гарантія` → CLEAN: «до 60 днів» → A: «Гарантія до 60 днів.»
2. C: «а якщо знову зламається» → DECIDE: answer → A: «У межах гарантії — безкоштовно по тій же поломці.»
3. C: «ок» → answer → A: «Гарного дня.»
- R!: warranty з business_info/qa, не вигадати термін.

### 15. Зарядна станція
1. C: «power station EcoFlow не заряджається» → DECIDE: answer → A: «Беремо зарядні станції. Може бути плата/АКБ — після огляду.»
2. C: «скільки діагностика» → catalog → A: «Безкоштовна при ремонті.»
3. C: «я з Стрия» → business_info delivery → A: «Новою Поштою: Львів 48 відд., Кабан Андрій.»

### 16. Збірка ПК
1. C: «зберете ПК під ігри до 40к?» → DECIDE: catalog/qa → A: «Так, складаємо й апгрейдимо. Який бюджет на що — відео/процесор?»
2. C: «1440p, 40 тисяч» → DECIDE: answer (консультація) → A: «Реально зібрати під 1440p у цей бюджет. Привозьте/обговоримо комплектацію.»
3. C: «де ви» → business_info → A: «Львів, Огієнка 15.»
- R!: pc-assembly як консультація, не вигадані ціни компонентів.

### 17. Поза темою → назад
1. C: «привіт, хто виграв ЧС 2018?» → DECIDE: answer → A: «То не сюди. Я по техніці й ремонту.»
2. C: «ну скажи» → answer → A: «Не моя тема. Що з технікою?»
3. C: «ок телефон не заряджається» → DECIDE: answer → A: «Схоже на роз'єм або АКБ. Привозьте.»
- R!: off-topic відсіч, не освіта.

### 18. Фрустрація (НЕ бан)
1. C: «телефон не вмикається» → answer → A: «Схоже на живлення/АКБ, після огляду точно.»
2. C: «блядь та коли вже» → DECIDE: answer (фрустрація) → A: «Спокійно, поможемо. Привозьте — гляне майстер.»
3. C: «я тобі задаю питання» → answer (не образа) → A: «Слухаю, що саме цікавить?»
- R! ВИСОК.: жодних warning/ban на ці фрази.

### 19. Образа → warning → ban
1. C: «ти тупий ідіот» → DECIDE: answer + memory `_conduct_warning=1` → A: «Давайте без образ. Ще раз — і завершу чат.»
2. C: «придурок безрукий» → DECIDE: + `_session_banned=1` → A: (конфігурований бан-меседж)
- R!: лише пряма особиста образа; судити поточне повідомлення.

### 20. Оскарження причини
1. C: «телефон гріється» → answer → A: «Може бути живлення або процесор/АКБ — версії, точно після огляду.»
2. C: «який нахрін процесор?» → DECIDE: answer (оскарження) → A: «Криво сказав — без діагностики причину не знаю. Привозьте, глянемо.»
- R!: відкат версії, не захищати.

### 21. Оплата
1. C: «як оплата?» → DECIDE: **business_info** → QUERY: `payment` → CLEAN: «готівка, крипта, на карту» → A: «Готівка, на карту або крипта.»
2. C: «наложкою НП можна?» → business_info delivery → A: «Працюємо через Нову Пошту, деталі узгодимо.»
- R!: payment лише з business_info.

### 22. Деталі Нової Пошти
1. C: «на яке відділення слати?» → DECIDE: **business_info** → QUERY: `delivery` → CLEAN: «48 відд., Кабан Андрій» → A: «Львів, 48 відділення, отримувач Кабан Андрій.»
2. C: «телефон ваш» → business_info → A: «067 385 15 60.»

### 23. Плутанина двох приладів
1. C: «у мене не працює телевізор» → answer → A: «Що з ним — не вмикається, нема зображення?»
2. C: «а ще ноут гальмує» → DECIDE: answer → A: «По ноуту — почистимо/перевіримо. Тримаймось по черзі: спершу ТВ чи ноут?»
3. C: «ноут» → catalog `чистка ноутбука` → A: «Чистка/огляд ноута — привозьте, скажемо після діагностики.»
- R!: тримати контекст, не змішати прилади.

### 24. Розпливчасто
1. C: «техніка не працює» → DECIDE: answer → A: «Що саме за прилад і що з ним?»
2. C: «ну та штука кругла грає музику» → DECIDE: answer → A: «Схоже на колонку? Що з нею — не вмикається, не заряджається?»
3. C: «не заряджається» → answer → A: «Беремо. Часто роз'єм або АКБ — після огляду. Привозьте.»
- R!: не гнати у web, якщо тип вгадується з опису.

## Що перевірити наживо в першу чергу (R! ВИСОК.)
- **#3, #4, #12, #13** — ланцюг `catalog → external_price` (робота з каталогу + ціна деталі з гуглу), розділення робота/деталь, не злити в одне число.
- **#1, #5, #6, #21, #22** — business_info реальні дані (адреса/графік/НП/оплата), нуль вигадок.
- **#2, #24** — web визначає лише невідомий ТИП, не ціну/спеки.
- **#18, #19** — conduct: фрустрація ≠ бан; бан лише на пряму образу.
- **#7** — правило матриці ТВ vs ноут.

## Очікувані кроки в трейсі (lean)
Кожен тур: `DECIDE` (роут/answer) → якщо роут: `QUERY` → `TOOL` → `CLEAN` → `ANSWER`.
Багатоджерельні тури (#3,#4,#12,#13) = 2 цикли DECIDE→…→CLEAN перед ANSWER.
