Глобальний Пайплайн Бота

1. Розпізнавання (Intent Router)

Користувач пише повідомлення. LLM аналізує текст і шукає співпадіння з тригерами зліва.

2. Специфічний пошук (Дія)

Запускається обрана дія (наприклад PRODUCT_SEARCH). Бот шукає точні дані (прайси, послуги) у SQL базі.

3. Глобальна База (Qdrant)

Якщо попередня нода нічого не знайшла (або якщо інтент не розпізнано), бот шукає відповідь у векторній базі Qdrant (мануали, інструкції).

4. Fallback (Остання лінія)

Якщо ніде немає відповіді, бот виконує Fallback Action (шукає в інтернеті, або просто вибачається перед клієнтом).
