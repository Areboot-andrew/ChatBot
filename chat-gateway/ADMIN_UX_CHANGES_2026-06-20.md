## Зміни

### 1. Перемикач активності КАТЕГОРІЙ каталогу
- Backend: `POST /admin/knowledge/prices/{cat_id}/toggle` — інвертує `ServiceCategory.enabled`.
- UI (`knowledge/prices.html`): крапка-статус + бейдж «вимкнена — бот не бачить» +
  кнопка «Вимкнути/Увімкнути» на кожній категорії; вимкнені притемнені.
- Навіщо: `search_catalog` і CONTENT MAP фільтрують `enabled==True`; вимкнена
  категорія невидима боту. Тепер це видно і керовано з UI.

### 2. Перемикач активності Q&A записів
- Backend: `POST /admin/knowledge/qa/{qa_id}/toggle` — інвертує `QaPair.enabled`.
- UI (`knowledge/index.html`, вкладка «Записи знань»): колонка «Статус»
  (активний/вимкнений) + кнопка «Вимкнути/Увімкнути»; вимкнені притемнені.
- Навіщо: `search_knowledge` фільтрує `enabled==True`.

### 3. Видалення записів (раніше можна було лише додати)
- Backend: `POST /admin/knowledge/qa/{qa_id}/delete` і
  `POST /admin/knowledge/prices/{cat_id}/delete` (категорія — каскадом з цінами).
- UI: кнопка «Видалити» з `confirm()` на Q&A записах і категоріях.
- Навіщо: оператор має чистити хибні/суперечливі записи, що вводять модель в оману.

### 4. Чистка суперечливих Q&A (міграція `35f0`)
Видаляє записи, що конфліктують з інтейк-гайдами або дублюються:
- графік роботи (конфлікт годин з `business_info`);
- 5 старих site-FAQ, що називають вузол причиною («причина у підсвітці/інверторі/
  акумуляторі») — суперечать гайдам «симптом не діагноз»;
- 1 дубль кавомашини.
Інтейк-гайди й прямі «так, ремонтуємо X» — лишені. Видалення по точному тексту
питання; оборотно через re-seed з ServicePage.tsx.

### 5. Поле «Embed-модель» (2026-06-21)
- Налаштування → нове поле «Embed-модель (для векторного пошуку / RAG)»
  (`meta.embed_model`). Точна назва embedding-моделі з LM Studio (напр.
  `text-embedding-embeddinggemma-300m`). Порожньо = вшитий дефолт `bge-m3`.
- `embed()` тепер приймає назву; прокинуто скрізь: `search_knowledge`,
  `process_and_vectorize_document`, route-пошук знань, дашборд-перевірка,
  завантаження документів. Embed timeout 3s→15s.
- Дашборд-картка «Embed-модель» тепер тестує саме обрану тенантом модель.
- Без міграції: поле опційне, дефолт працює; власник вписує назву в панелі.
