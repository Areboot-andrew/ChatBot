# Щоденник змін (Changelog)

У цьому файлі ми ведемо історію всіх важливих архітектурних рішень, фіксів та нових фіч.

## [2026-06-12] - Оптимізація парсингу та стабілізація LLM з'єднань

### Додано
- **UI Авто-визначення провайдера:** У `settings.html` додано JS-скрипт, який при завантаженні сторінки аналізує збережений `base_url` і автоматично виставляє правильний шаблон у випадаючому списку (Local, Groq, OpenRouter тощо).
- **Фонова перевірка з'єднання:** При відкритті `settings.html` система автоматично робить тестовий запит (пінг) до обраної LLM моделі і одразу показує статус з'єднання (✔ Активне або ✖ Помилка).
- **Кнопка "Перевірити з'єднання":** Додано ручну кнопку для тестування підключення до LLM без необхідності заходити в пісочницю.
- **Deep Web Parsing (Глибокий парсинг):** Інструмент `search_internet` тепер не просто бере короткі сніпети з DuckDuckGo, а фізично переходить на перший найрелевантніший знайдений сайт і витягує весь його текстовий контент (через `trafilatura`) для максимальної точності технічних відповідей.

### Виправлено
- **Блокування Event Loop:** Пошук в інтернеті (`search_internet`) обгорнуто в `asyncio.to_thread`, щоб запобігти зависанню асинхронного FastAPI-воркера під час HTTP-запитів.
- **Зникнення контексту в Роутері:** Виправлено баг, через який маршрутизатор не бачив історію розмови, якщо клієнт писав короткі фрази (напр. "Biostar A78MD"). Тепер роутер обов'язково дивиться в історію, що гарантує спрацювання `WEB_SEARCH` замість відписок.
- **Обрізання довгих відповідей (max_tokens bug):** Стрімінговий генератор фінальної відповіді раніше ігнорував налаштування `max_tokens` з бази і використовував жорсткий ліміт (1024), через що моделі могли видавати порожні повідомлення. Тепер ліміт чесно читається з БД.
- **Зламаний DuckDuckGo:** Видалено глючну бібліотеку `duckduckgo_search` (яку часто блокують). Написано власний легкий HTML-скрапер для `html.duckduckgo.com`, який працює стабільно і швидко.
- **Дефолтна адреса LM Studio:** Змінено `192.168.1.84:1234` на універсальну адресу `host.docker.internal:1234` у `config.py` для коректної роботи Docker-контейнера з Windows/Mac хостами.
- **Відображення логів (Трейси):** Покращено деталізацію логів у Пісочниці. Тепер чітко видно етап "Маршрутизації" та реальний сервер LLM, до якого йде запит (замість пустого рядка).
